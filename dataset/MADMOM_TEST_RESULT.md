# madmom 和弦识别测试结果

## 测试环境
- Python: 3.10
- madmom: 0.16.1
- 测试文件: `dataset/vevo_audio/wav/001.wav`

## 测试结果

### 1. madmom 成功运行
- ✓ 成功安装并修复了兼容性问题
- ✓ 成功识别出和弦

### 2. 识别的和弦类型
**结果：madmom 也只识别出了 `maj` 和 `min` 两种属性**

- 识别到的和弦类型：`F:maj`, `G:maj`, `E:min`, `A:min`, `C:maj`, `D#:maj`, `G#:maj`, `F:min`, `A#:maj`, `A:maj`, `E:maj`, `G:min`, `D:min`
- 属性类型：只有 `maj` 和 `min`

### 3. 与 Omnizart 对比

| 工具 | 识别的属性类型 | 和弦总数 |
|------|--------------|---------|
| Omnizart | maj, min | 25 种 |
| madmom | maj, min | 类似（取决于音频） |

### 4. 结论

**madmom 的默认模型（CRFChordRecognitionProcessor）也只支持识别 `maj` 和 `min` 两种属性。**

这可能是因为：
1. **训练数据限制**：madmom 的模型训练时只使用了包含 maj/min 的数据集
2. **模型设计**：为了简化任务，只区分大三和弦和小三和弦
3. **识别精度**：更细粒度的和弦识别需要更复杂的模型和更多的训练数据

### 5. 建议

1. **接受限制**：继续使用 Omnizart 或 madmom，只识别 maj/min
   - 对于大多数流行音乐，maj/min 已经足够
   - 简单、快速、稳定

2. **使用官方数据集**：如果官方数据集有更丰富的和弦标注，直接使用官方数据

3. **后处理增强**：基于 MIDI 或音频特征推断更复杂的和弦类型
   - 需要额外开发
   - 精度可能不高

4. **使用专业工具**：如果需要更丰富的和弦类型，考虑使用专业音乐分析软件或人工标注
